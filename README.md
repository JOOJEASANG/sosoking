# 소소킹 판결소

사소한 한 줄 사건을 AI가 과하게 진지한 재판/판결문 형식으로 처리하는 Firebase 기반 오락 서비스입니다.

소소킹의 핵심은 **뉴스나 위원회가 아니라 재판과 판결**입니다. 모기, 라면, 리모컨, 마지막 만두, 엘리베이터 같은 별것 아닌 일을 마치 중대한 사건처럼 접수하고, AI 재판부가 엄숙한 판결문으로 웃기게 마무리합니다.

---

## 프로젝트 기준

소소킹을 수정할 때는 아래 기준을 우선합니다.

### 1. 핵심 컨셉

- 소소킹은 **사소한 한 줄 사건을 과하게 진지하게 판결하는 AI 오락 서비스**입니다.
- 사용자가 입력하는 내용은 꼭 사람 간 분쟁일 필요가 없습니다.
- 예시:
  - 오늘 모기 때문에 잠을 못 잠
  - 엘리베이터가 내 앞에서 닫힘
  - 마지막 만두를 누가 먹어야 하는가
  - 리모컨은 잡은 사람이 임자인가
  - 치약이 양치 중에 끝남
- 핵심 재미는 **하찮은 사건 + 과한 재판 말투 + 말도 안 되게 사소한 형량**에서 나와야 합니다.
- 뉴스/속보 느낌은 보조 장치로만 사용하고, 메인 정체성은 항상 **재판/판결**입니다.

### 2. 금지되는 방향

아래 방향으로 사이트를 바꾸지 않습니다.

- 뉴스 사이트처럼 바꾸기
- 위원회/상황실 컨셉을 메인으로 만들기
- 실제 법률 서비스처럼 보이게 만들기
- 정치, 혐오, 성적 내용, 실제 범죄, 개인정보 중심 콘텐츠로 확장하기
- 너무 장황하거나 진짜 법률문처럼 무겁게 만들기
- 웃기려고 욕설, 비하, 자극적인 표현에 의존하기

### 3. 결과물 기본 구조

AI 결과와 화면 구성은 되도록 아래 구조를 따릅니다.

1. **소장 접수** — 한 줄 사건을 재판에 올림
2. **증거조사** — 하찮은 증거를 진지하게 채택
3. **쟁점 정리** — 별것 아닌 일을 판결 쟁점으로 정리
4. **판결이유** — AI 재판부가 과하게 엄숙하게 판단
5. **주문** — 최종 판결
6. **소소 형량** — 실행 가능하지만 매우 하찮은 처분
7. **웃김 점수** — 유저가 1~10점으로 평가

### 4. 문구 톤앤매너

소소킹 문구는 아래 톤을 유지합니다.

- 진지한 척하지만 내용은 하찮게
- 재판부, 원고, 피고, 증거, 주문, 형량 같은 재판 단어를 적극 활용
- “피고 모기”, “이불 방어선”, “마지막 만두 증거물”처럼 사소한 대상을 재판 대상으로 세움
- 문장은 단호하고 엄숙하게 쓰되, 결론은 가볍고 웃기게 마무리

예시 톤:

```text
피고 모기는 원고의 귓가 30cm 이내 접근을 오늘 밤 1회 금지한다.
```

```text
재판부는 베개를 세 번 뒤집은 행위를 수면권 방어를 위한 정당한 긴급조치로 본다.
```

## AI 생성 기준

AI 결과 생성 프롬프트는 다음 원칙을 따릅니다.

- 목표 웃김 강도는 높게 잡되, 억지 개그를 남발하지 않습니다.
- 입력이 하찮을수록 결과는 더 엄숙하게 씁니다.
- 실제 법률 조언처럼 보이지 않도록 반드시 오락 콘텐츠임을 전제로 합니다.
- 결과에는 재판적 장면이 있어야 합니다.
  - 예: 재판부, 피고, 원고, 증거, 방청석, 판결이유, 주문, 형량
- 마지막 형량은 반드시 가볍고 실행 가능한 소소한 내용이어야 합니다.

좋은 결과 예시 방향:

```text
입력: 오늘 모기 때문에 잠을 못 잠

피고 모기는 새벽 시간대 원고의 귓가를 반복 비행하여 수면권을 흔든 점이 인정된다.
재판부는 베개를 뒤집은 횟수와 원고의 멍한 표정을 증거로 채택한다.
피고 모기는 오늘 밤 원고 귓가 30cm 이내 접근을 금한다.
```

## 웃김 점수 기준

소소킹은 유저들이 공개 판결기록을 보고 1~10점으로 웃김 점수를 매길 수 있는 방향을 기준으로 합니다.

- 점수는 공개 판결기록의 랭킹 요소입니다.
- 단순 좋아요보다 “얼마나 웃겼는지”를 보여주는 것이 목적입니다.
- 랭킹명은 “소소킹 후보”, “오늘의 소소킹”, “웃김 점수 TOP” 같은 표현을 사용합니다.

## 구조

- `public/` — Firebase Hosting 정적 프론트엔드
- `public/js/pages/` — 홈, 접수, 재판 진행, 결과, 정책, 내 사건 페이지
- `public/admin/` — 관리자 페이지
- `functions/` — Firebase Cloud Functions, Gemini API 호출
- `firestore.rules` — Firestore 접근 제어 규칙
- `firestore.indexes.json` — Firestore 복합 인덱스

---

## 주요 보안 구조

- 사용자는 Firebase Auth로 접속합니다.
- 사건 접수는 클라이언트 직접 쓰기가 아니라 `submitCase` Callable Function에서 처리합니다.
- 일일 접수 제한과 쿨다운은 서버 트랜잭션으로 검사합니다.
- `generateTrial`은 로그인 여부와 사건 소유자를 검증한 뒤 실행합니다.
- `results` 문서는 소유자, 공개 판결문, 관리자만 읽을 수 있습니다.
- 사용자 입력 및 AI 생성 결과는 화면 렌더링 전 escape 처리합니다.

---

## 관리자 설정

관리자 권한은 Firestore `admins/{uid}` 또는 `admins/{email}` 문서 존재 여부로 판단합니다.

관리자 페이지에서 설정 가능한 항목:

- 일일 접수 한도
- 재접수 쿨다운
- 금칙어
- AI 자동 생성 설정
- AI 프롬프트/모델 설정
- 사용량·비용 계산 기준
- 사업자 정보
- 정책 문서

관리자 AI 자동 생성도 반드시 이 README의 프로젝트 기준을 따라야 합니다.

## 배포

```bash
firebase deploy
```

Functions 배포 전 Gemini API Key Secret이 필요합니다.

```bash
firebase functions:secrets:set GEMINI_API_KEY
```

프론트와 Functions를 함께 되돌리거나 수정한 경우에는 아래처럼 배포합니다.

```bash
firebase deploy --only functions,hosting
```

## 주의

본 서비스는 AI 기반 오락 콘텐츠입니다. 생성된 판결문은 실제 법률 자문이나 법원 판결이 아니며 어떠한 법적 효력도 없습니다.

실제 법률 문제, 범죄 피해, 학교폭력, 가정폭력, 의료·정신건강 문제 등 중대한 사안은 소소킹에서 다루는 대상이 아니며, 관련 기관이나 전문가의 도움을 받아야 합니다.
