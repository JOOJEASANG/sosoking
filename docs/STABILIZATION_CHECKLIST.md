# 소소킹 안정화 체크리스트

## 목적

여러 단계의 긴급 수정과 UI 보정이 누적된 상태에서, 배포 후 다시 빈 화면·모듈 로드 실패·모바일 깨짐·관리자 기능 오류가 발생하지 않도록 확인 기준을 고정한다.

## 배포 전 확인

### 1. 필수 배포 범위

관리자 회원현황, 데이터 관리, callable functions 변경이 포함되어 있으므로 다음 배포가 필요하다.

```bash
firebase deploy --only hosting,functions --project sosoking-481e6
```

functions 변경이 없고 화면 CSS/JS만 확인할 때만 hosting 단독 배포를 사용한다.

```bash
firebase deploy --only hosting --project sosoking-481e6
```

### 2. 캐시 확인

- `public/sw.js` 캐시 버전이 최신인지 확인한다.
- JS/CSS/HTML/manifest는 서비스워커 캐시에 저장하지 않는다.
- 배포 후 문제가 남으면 브라우저에서 Service Worker unregister 후 새로고침한다.

### 3. 관리자 권한 확인

- 현재 로그인 UID가 `admins/{uid}` 문서에 존재해야 한다.
- 관리자 데이터 관리 callable은 관리자 문서가 없으면 권한 오류가 난다.

## 배포 후 화면 확인

### 홈

- 비회원 상태에서 사이트 목적이 바로 보이는지 확인
- 회원 상태에서 닉네임 문구가 보이는지 확인
- 최근 피드가 카드형이 아니라 얇은 리스트형인지 확인
- 바로 만들기 버튼들이 모바일에서 가로 스크롤되는지 확인

### 피드

- 피드가 10개씩 보이는지 확인
- 페이지 번호/이전/다음이 동작하는지 확인
- 최신순/인기순/댓글순/조회순/참여순 드롭다운이 동작하는지 확인
- 검색이 제목, 본문, 태그, 유형명에 반응하는지 확인
- 검색 결과 요약이 표시되는지 확인

### 글쓰기

- 일반글 작성 가능
- 투표/판정 작성 가능
- 빈칸 채우기에서 `___가 ___했다`, `3, 4` 방식 저장 가능
- 미친작명소 작성 가능
- 삼행시 작성 가능
- 막장릴레이 작성 가능
- 미친퀴즈 작성 가능
- 익명비밀글 작성 가능
- 모바일에서 입력창이 확대/겹침 없이 보이는지 확인

### 상세

- 투표/판정 투표 가능
- 빈칸 채우기 여러 빈칸이 따로 보이는지 확인
- 빈칸 채우기 등록값이 `빈칸 1: ... / 빈칸 2: ...` 형태인지 확인
- 참여글/답글에 회원 닉네임이 표시되는지 확인
- 비회원 상태에서 로그인 유도 문구가 보이는지 확인

### 관리자

- 사이트 홈으로 버튼이 사라졌는지 확인
- 대시보드 통계가 표시되는지 확인
- 게시물 숨김/공개가 동작하는지 확인
- 게시물 삭제 시 `삭제` 입력 확인이 뜨는지 확인
- 신고 관리에서 게시물 숨김/처리완료가 동작하는지 확인
- 회원 현황이 functions 배포 후 표시되는지 확인
- AI 관리가 최소 AI 설정 중심으로 보이는지 확인
- 데이터 관리 탭이 보이는지 확인
- 데이터 관리 삭제 시 `삭제` 입력 확인이 뜨는지 확인

### 모바일

- 하단 네비가 `홈 / 피드 / 작성 / 게임 / 내정보`인지 확인
- 가운데 작성 버튼이 글쓰기 화면으로 이동하는지 확인
- 하단 네비가 콘텐츠를 가리지 않는지 확인
- iOS Safari 하단 safe-area에서 버튼이 잘리지 않는지 확인
- Android Chrome 주소창 접힘/펼침 상태에서 하단 네비가 흔들리지 않는지 확인

## 문제 발생 시 우선 확인 순서

1. 개발자도구 Console 에러 확인
2. Network에서 실패한 JS/CSS 모듈 확인
3. Service Worker unregister 후 새로고침
4. Firebase Hosting 배포가 최신 commit인지 확인
5. functions callable 문제면 functions 배포 로그 확인
6. Firestore 권한 문제면 `admins/{uid}` 문서 확인

## 장기 정리 원칙

- 현재 safe/fix/patch 파일들은 운영 안정화를 위해 유지한다.
- 바로 삭제하지 않는다.
- 다음 리팩터링 때 같은 기능끼리 원본 파일에 통합한다.
- 통합 전에는 반드시 배포 전후 체크리스트를 반복한다.
