# Security hardening

이 문서는 2026-07-17 보안 강화 변경의 운영 영향과 배포 후 확인 항목을 정리합니다.

## 적용된 변경

- 이메일/비밀번호 계정은 이메일 인증 완료 후 AI·프로필·소셜 기능 사용
- 문서 ID 경로 구분자 차단
- 사건 접수·사건명 추천 실패 시 사용량 예약 환불 및 중단 예약 자동 복구
- 사건 이미지 실제 파일 시그니처 검사
- 판결 저장과 사건 완료 상태 원자적 커밋
- 공개/비공개 전환 서버 트랜잭션화 및 공개 전 개인정보 재검사
- 댓글의 Firebase UID 공개 제거
- 항소심 횟수 제한, 동시 실행 잠금, 로컬 대체 판결
- 오늘의 AI 판결 날짜별 실행 잠금
- 프로필 사진 경로를 사용자별 `avatar.jpg`로 고정
- Firestore 핵심 데이터 클라이언트 직접 쓰기 차단
- Hosting 기본 보안 헤더 적용

## 배포 후 필수 확인

1. 이메일 인증 메일 발송 도메인과 템플릿을 Firebase Console에서 확인합니다.
2. Storage Rules는 별도 수동 워크플로로 배포합니다.
3. 기존에 여러 이름으로 저장된 프로필 사진은 새 사진 저장 시 `avatar.jpg`로 교체됩니다. 과거 파일은 관리자가 별도로 정리할 수 있습니다.
4. 공개 전환이 개인정보 탐지로 차단될 때 실제 개인정보를 제거한 후 다시 시도합니다. 탐지는 안전을 우선하므로 일부 일반 문구가 차단될 수 있습니다.
5. Firebase App Check는 콘솔에서 웹 앱용 공급자와 사이트 키를 발급한 후 별도 적용해야 합니다. 저장소만으로 유효한 사이트 키를 생성할 수 없어 이번 변경에는 강제 적용하지 않았습니다.
6. Google Cloud Billing 예산 알림과 Gemini 사용량 경보를 별도로 설정합니다.

## 롤백 주의사항

Firestore Rules만 이전 버전으로 되돌리면 브라우저 직접 쓰기가 다시 열릴 수 있습니다. Functions, Firestore Rules, Hosting을 같은 커밋 기준으로 함께 배포해야 합니다.
