# 소소킹 전체 UX·운영 점검 계획

## 목표

회원, 비회원 접속자, 관리자/운영자 관점에서 소소킹의 화면 구성, 글쓰기 흐름, 참여 흐름, 관리자 운영 흐름, 모바일 레이아웃, 데이터 관리 안정성을 단계별로 점검하고 개선한다.

## 1단계: 관리자 기본 UI 정리 — 완료

- 관리자 페이지의 중복 이동 버튼 제거
- 관리자 메뉴 구조 확인
- 모바일 관리자 메뉴 가로 스크롤 안정화
- 관리자 데이터 관리 탭 접근성 확인

## 2단계: 회원/접속자 관점 점검 — 진행 완료

완료 항목:

- 홈 첫인상을 간결하게 정리
- 회원 상태에서는 닉네임 기반 문구 표시
- 비회원 상태에서는 사이트 목적이 바로 보이도록 문구 정리
- 최근 피드는 카드형 대신 얇은 리스트형으로 정리
- 바로 만들기 유형을 현재 실제 글쓰기 유형 기준으로 정리
- 인기 피드/최근 피드의 유형 표기 통일
- 댓글 작성 영역 문구 개선
- 비회원 댓글/참여 유도 문구 개선
- 댓글 작성 시 닉네임 fallback 보강
- 관련 UI 스타일 보정

남은 확인 항목:

- 실제 배포 후 모바일 홈 첫 화면 높이 확인
- 로그인 전후 홈 문구 정상 전환 확인
- 댓글 등록 후 새로고침 없이 목록 갱신하는 개선은 추후 안정화 단계에서 검토

## 3단계: 글쓰기/참여 기능 점검 — 다음 진행 대상

- 일반글
- 투표/판정
- 빈칸 채우기
- 미친작명소
- 삼행시
- 막장릴레이
- 미친퀴즈
- 익명비밀글

각 유형별로 작성 화면, 저장 데이터, 피드 카드 표기, 상세 참여 화면, 댓글/답글, 수정 화면을 확인한다.

## 4단계: 피드/검색/정렬 점검

- 피드 10개 페이지네이션
- 최신순/인기순/댓글순/조회순/참여순 드롭다운
- 유형 필터
- 검색 결과 UI
- 빈 결과 UI
- 모바일 가로 스크롤/버튼 위치

## 5단계: 관리자/운영자 관점 점검

- 대시보드 통계
- 게시물 관리
- 신고 관리
- 회원 현황
- AI 관리
- 데이터 관리
- 삭제/숨김/복구 가능성
- 실수 방지 문구와 confirm 문구

## 6단계: 모바일 레이아웃 점검

- 홈
- 피드
- 글쓰기
- 상세
- 계정
- 관리자
- 데이터 관리
- 하단 네비게이션/푸터 간섭 여부

## 7단계: 정리 및 안정화

- 임시 safe 파일/보정 파일 중 장기 유지할 것과 통합할 것 분류
- 중복 스크립트 제거 후보 정리
- 서비스워커 캐시 문제 점검
- 배포 후 확인 체크리스트 작성
