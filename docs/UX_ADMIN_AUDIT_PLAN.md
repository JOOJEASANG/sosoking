# 소소킹 전체 UX·운영 점검 계획

## 목표

회원, 비회원 접속자, 관리자/운영자 관점에서 소소킹의 화면 구성, 글쓰기 흐름, 참여 흐름, 관리자 운영 흐름, 모바일 레이아웃, 데이터 관리 안정성을 단계별로 점검하고 개선한다.

## 1단계: 관리자 기본 UI 정리 — 완료

- 관리자 페이지의 중복 이동 버튼 제거
- 관리자 메뉴 구조 확인
- 모바일 관리자 메뉴 가로 스크롤 안정화
- 관리자 데이터 관리 탭 접근성 확인

## 2단계: 회원/접속자 관점 점검 — 완료

완료 항목:

- 홈 첫인상을 간결하게 정리
- 회원 상태에서는 닉네임 기반 문구 표시
- 비회원 상태에서는 사이트 목적이 바로 보이도록 문구 정리
- 최근 피드는 카드형 대신 얇은 리스트형으로 정리
- 바로 만들기 유형을 현재 실제 글쓰기 유형 기준으로 정리
- 인기 피드/최근 피드의 유형 표기 통일
- 댓글 작성 영역 문구 개선
- 비회원 댓글/참여 유도 문구 개선
- 댓글 작성 시 닉네임 fallback 보강
- 관련 UI 스타일 보정

## 3단계: 글쓰기/참여 기능 점검 — 완료

- 일반글
- 투표/판정
- 빈칸 채우기
- 미친작명소
- 삼행시
- 막장릴레이
- 미친퀴즈
- 익명비밀글

완료 항목:

- 빈칸 채우기 다중 빈칸 제출값 분리 저장 보정
- 빈칸 채우기 작성 화면을 `___가 ___했다` 방식에 맞게 개선
- 빈칸별 칸 수를 `3, 4`처럼 입력할 수 있게 개선
- 투표/판정 문구를 댓글 토론 가능 구조로 정리
- 익명 유형 명칭을 익명비밀글로 정리
- 수정 화면에서 빈칸 채우기 설정이 보이지 않던 문제 보완
- 수정 화면에서 기존 참여 기능 라벨과 설정이 더 정확히 보이도록 보완
- 수정 저장 시 빈칸 채우기 `blankCounts`, `blanks`, `charCount`, `blankCount` 구조 유지

## 4단계: 피드/검색/정렬 점검 — 완료

완료 항목:

- 피드 10개 페이지네이션 유지
- 최신순/인기순/댓글순/조회순/참여순 드롭다운 유지
- 검색 범위 확대: 제목뿐 아니라 본문, 태그, 작성자, 유형명까지 검색
- 검색 결과 수, 현재 페이지, 정렬 상태 요약 UI 추가
- 검색 중에는 유형 필터가 자동 해제된다는 안내 추가
- 빈 검색 결과에서 전체 피드로 돌아가는 버튼 추가
- 참여순 점수에 댓글, 반응, 참여 카운트를 함께 반영
- 모바일에서 결과 요약과 페이지네이션 버튼이 눌리기 쉽도록 보정

남은 확인 항목:

- Firestore 클라이언트 검색은 최근 300개 기준이므로 글이 많아지면 서버 검색 또는 인덱스 기반 검색 검토 필요
- 페이지네이션은 현재 클라이언트 캐시 기준이므로 대량 데이터에서는 서버 페이지네이션 개선 검토 필요

## 5단계: 관리자/운영자 관점 점검 — 완료

완료 항목:

- 관리자 사이드바의 사이트 홈 이동 버튼 원본 제거
- 대시보드 통계 보강: 총 게시물, 오늘 새 글, 미처리 신고, 회원 문서, 숨김 게시물
- 대시보드에 운영 체크 안내 추가
- 최근 게시물에 유형/작성자 표기 추가
- 게시물 관리에 유형, 작성일, 문서 ID, 수정 버튼 추가
- 게시물 숨김 처리 시 hiddenAt, hiddenBy 기록
- 삭제는 단순 confirm 대신 `삭제` 입력 확인 방식으로 강화
- 신고 관리에서 신고된 게시물 숨김 버튼 추가
- 신고 처리 시 resolvedAt, resolvedBy 기록
- 회원 현황을 게시글 작성자 임시 집계 대신 getAdminMemberList 함수 기반으로 정리
- AI 관리는 최소 AI 사용 여부 중심으로 정리
- 데이터 관리 탭에 삭제 주의 안내 추가
- 데이터 관리 삭제도 `삭제` 입력 확인 방식으로 강화
- 관리자 모바일 테이블, 액션 버튼, 운영 안내 스타일 보정

## 6단계: 모바일 레이아웃 점검 — 완료

완료 항목:

- 모바일 하단 네비게이션을 `홈 / 피드 / 작성 / 게임 / 내정보` 구조로 정리
- 기존 가운데 피드 버튼이 글쓰기처럼 보이던 혼동 제거
- 작성 버튼은 실제 글쓰기 화면으로 이동하도록 수정
- 모바일 전역 폭넘침 방지 보정 추가
- 하단 네비 fixed 처리 및 safe-area 보정
- 푸터와 하단 네비 간섭 방지
- 홈 히어로, 통계, 바로 만들기, 최근 피드 모바일 간격 보정
- 피드 검색, 필터, 정렬, 페이지네이션 모바일 터치 영역 보정
- 글쓰기 유형 선택, 입력폼, 하단 버튼 모바일 보정
- 상세 제목, 본문, 반응 버튼, 참여 기능 영역 모바일 보정
- 계정/로그인/설정 카드 최대폭 보정
- 관리자/데이터 관리 모바일 메뉴, 테이블, JSON 영역 보정
- 초소형 모바일 374px 이하 보정 추가

## 7단계: 정리 및 안정화 — 완료

완료 항목:

- 서비스워커 캐시 버전을 `sosoking-v7-2026-05-20-stable`로 갱신
- HTML/JS/CSS/JSON/manifest는 서비스워커 캐시에 저장하지 않고 항상 최신 네트워크 응답을 쓰도록 정리
- Firebase/Auth/Firestore/Functions/Kakao 외부 요청은 서비스워커 캐시에서 제외
- 이미지/아이콘 등 정적 자산만 캐시하도록 명확화
- 배포 후 확인용 `docs/STABILIZATION_CHECKLIST.md` 추가
- 임시 보정 파일 분류용 `docs/TEMP_PATCH_INVENTORY.md` 추가
- safe/fix/patch 파일은 즉시 삭제하지 않고 장기 통합 후보로 문서화
- CSS 중복 파일도 즉시 삭제하지 않고 장기 통합 후보로 문서화

최종 배포 권장 명령:

```bash
firebase deploy --only hosting,functions --project sosoking-481e6
```

배포 후 확인 문서:

- `docs/STABILIZATION_CHECKLIST.md`
- `docs/TEMP_PATCH_INVENTORY.md`

장기 과제:

- 글쓰기/수정 스크립트 통합
- 상세 참여 스크립트 통합
- 관리자 탭 모듈 구조화
- 모바일 CSS 통합
- 서버 검색 또는 인덱스 기반 검색 도입 검토
- 게시물 삭제 대신 휴지통/복구 구조 도입 검토
