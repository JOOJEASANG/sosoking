# 소소킹 가상현실 정치게임 점검 및 보완 설계

## 목표

소소킹은 단순 커뮤니티나 AI 도구 모음이 아니라, 사용자가 `소소공화국`이라는 가상 국가 안에서 정당에 가입하고 정치력을 쌓아 정당전, 대선, 탄핵, 언론, 여론 흐름에 참여하는 가상 정치 시뮬레이션 사이트로 운영한다.

핵심 느낌은 다음과 같다.

- 실제 정치처럼 보이지만 현실 정당·현실 인물과 직접 연결하지 않는다.
- 너무 가볍지도, 너무 무겁지도 않게 운영한다.
- 사용자는 매일 접속했을 때 오늘의 정세, 우리 당 순위, 대통령 상태, 여론 흐름을 바로 이해해야 한다.
- 모든 활동은 정치력과 정당 세력에 연결되어야 한다.

## 현재 상태 요약

### 이미 갖춘 축

- 회원/닉네임/포인트 구조
- 피드와 댓글
- 정당 가입
- 정당 정치력
- 정치배틀
- 대통령 선거
- 대통령 포고령
- 탄핵 청원
- 소소신문
- 주간 정당전 보상
- 정치력 랭킹
- 역대 당선자

### 보완이 필요한 축

- 국가 상태창이 부족하다.
- 대통령 지지율이 약하다.
- 여당/야당 구도가 약하다.
- 국회 역할이 약하다.
- 헌법재판소 역할이 일반 판결과 섞여 있었다.
- 일부 메뉴명이 AI 도구처럼 보여 정치게임 몰입을 깨고 있었다.
- 7당 구조는 유저가 적을 때 정당이 비어 보일 가능성이 크다.
- 일부 확장 UI는 렌더링 타이밍에 따라 사라질 수 있었다.

## 확정 방향

### 정당 구조

장기 운영은 3당 체제를 권장한다.

1. 국민안정당
   - 안정, 질서, 경험, 행정 능력
   - 여당 이미지에 가장 잘 맞음

2. 청년혁명당
   - 개혁, 변화, 온라인 여론, 젊은 정치
   - 제1야당 이미지에 가장 잘 맞음

3. 중도민주당
   - 실용, 데이터, 조정, 여론 균형
   - 캐스팅보트 이미지에 가장 잘 맞음

현재 7당 데이터는 바로 삭제하지 말고, 신규 노출과 신규 입당부터 3당 중심으로 전환한다.

### 기관 역할

#### 대통령실

- 대통령 포고령
- 대통령 공약
- 대통령 지지율
- 대통령 업적
- 대통령 위기 대응

#### 국회

- 법안 발의
- 정당별 찬반
- 대통령 견제
- 탄핵소추 표결

#### 헌법재판소

헌법재판소는 일반 판결소가 아니다.

- 평상시 일반 메뉴에서 강하게 노출하지 않는다.
- 탄핵소추가 통과된 경우에만 핵심 화면으로 등장한다.
- 탄핵 인용/기각을 결정한다.
- 인용 시 대통령 파면 및 조기 대선 흐름으로 간다.

#### 국민신문고

기존 AI 판결소의 일반 사건 기능은 국민신문고로 운영한다.

- 시민 민원
- 정책 불만
- 정당 논쟁
- 소소신문 편향 제보
- 포고령 불편 신고

즉, 국민신문고는 일반 민원 처리, 헌법재판소는 탄핵 전용이다.

## 사용자가 매일 들어오는 이유

사용자는 정치 용어 자체 때문에 들어오지 않는다. 다음 질문에 대한 답을 보기 위해 들어온다.

- 오늘 소소공화국에 무슨 일이 생겼나?
- 우리 당이 1위인가?
- 대통령 지지율은 올랐나 떨어졌나?
- 오늘 투표하면 우리 당에 도움이 되나?
- 탄핵 분위기가 생겼나?
- 소소신문에 우리 당이 어떻게 나왔나?

따라서 홈 화면은 다음 순서가 좋다.

1. 국가 상태창
2. 대통령 현황
3. 오늘의 정세
4. 우리 당 현황
5. 오늘 할 일
6. 소소신문
7. 정치배틀

## 국가 상태창 설계

홈 상단에 다음 데이터를 표시한다.

- 대통령
- 여당
- 대통령 지지율
- 경제
- 치안
- 복지
- 교육
- 언론 신뢰도
- 국정 안정도
- 국가 등급

예시:

```text
소소공화국 국가 현황
대통령: 홍길동
여당: 국민안정당
지지율: 47%
경제: 72
치안: 65
복지: 54
교육: 61
국정 안정도: B+
```

## 정치 이벤트 설계

매일 하나의 정치 이벤트가 있으면 사이트가 살아 보인다.

예시:

- 물가 상승 논란
- 청년 지원금 공약 후퇴
- 여당 내부 갈등
- 야당 장외투쟁
- 대통령 포고령 논란
- 소소신문 편향 논란
- 정당전 여론조작 의혹

각 이벤트는 너무 현실 정당과 닮지 않게, 그러나 실제 정치처럼 보여야 한다.

이벤트 구조:

```text
제목
상황 설명
관련 수치 변화
정당별 입장
유저 투표
소소신문 후속 기사
```

## 탄핵 흐름 설계

탄핵은 다음 4단계로 운영한다.

1. 탄핵 청원
   - 대통령 지지율 하락 또는 포고령 비판 증가 시 노출

2. 국회 탄핵소추 표결
   - 유저 투표 또는 정당 정치력 기준으로 찬반 계산

3. 헌법재판소 심판
   - 탄핵소추가 통과되면 개방
   - 재판관 9인 심판 콘셉트
   - 인용/기각 결정

4. 결과
   - 인용: 대통령 파면, 조기 대선
   - 기각: 대통령 복귀, 야당 역풍, 지지율 변화

## 메뉴 정리 기준

핵심 메뉴:

- 홈
- 정당
- 대선
- 국회
- 대통령실
- 소소신문
- 정치배틀
- 랭킹
- 국민신문고
- 내 정보

조건부 메뉴:

- 헌법재판소: 탄핵 국면에서만 강하게 노출

축소 또는 숨김 대상:

- AI킹 같은 도구식 명칭
- 단순 재미성 판결소 명칭
- 정치 루프와 직접 관련 없는 일반 미니게임

## 디자인 점검 기준

각 화면은 다음 요소 중 최소 2개 이상을 가져야 한다.

- 기관명: 대통령실, 국회, 헌법재판소, 선거관리위원회, 소소신문
- 상태 수치: 지지율, 정치력, 의석, 여론, 안정도
- 정치적 맥락: 여당/야당, 정당 입장, 정책 효과
- 오늘의 행동: 투표, 유세, 민원, 보상, 기사 읽기

없으면 단순 게시판이나 AI 도구처럼 보인다.

## 다음 개발 우선순위

### 1단계: 세계관 정리

- 판결소 → 국민신문고
- 헌법재판소는 탄핵 전용 문구로 분리
- AI킹 → 대통령실 또는 정치참모실 명칭 검토
- 푸터/사이드바/하단 메뉴 문구 통일

### 2단계: 국가 상태창

- 국가 상태 데이터 구조 생성
- 홈 상단에 국가 현황 카드 추가
- 대통령 지지율 표시
- 여당/야당 구도 표시

### 3단계: 3당 체제 전환

- 신규 노출 정당 3개로 제한
- 대선 후보 3당 중심으로 생성
- 정당전 보상도 3당 기준으로 전환
- 기존 7당 데이터는 관리자용 보관 또는 숨김 처리

### 4단계: 탄핵 시스템 강화

- 탄핵 청원
- 국회 탄핵소추 표결
- 헌법재판소 심판
- 조기 대선

### 5단계: 국회 시스템 추가

- 법안 발의
- 정당별 찬반
- 의석수 또는 정치력 기반 가결/부결
- 소소신문 기사화

### 6단계: 소소신문 강화

- 오늘의 정세 요약
- 대통령실 반응
- 여당 논평
- 야당 비판
- 국민 여론
- 정당전 판세

## 최근 보완 사항

- 주간 정당전 카드가 정당 페이지 렌더링 후 안정적으로 삽입되도록 보정했다.
- 정당전 보상 수령 시 1위 정당을 트랜잭션 안에서 다시 계산하도록 보정했다.
- 일반 판결소 화면은 국민신문고로 정리했다.
- 헌법재판소는 탄핵 전용 기관으로 보이도록 문구 보정 모듈을 추가했다.

## 운영 원칙

소소킹의 재미는 정답 맞히기가 아니라 `정세가 매일 바뀌는 느낌`이다.

따라서 모든 기능은 다음 질문 중 하나에 답해야 한다.

- 우리 당에 도움이 되는가?
- 대통령 권력이 변하는가?
- 여론이 움직이는가?
- 신문에 기록되는가?
- 다음 선거에 영향을 주는가?

위 질문에 답하지 못하는 기능은 축소하거나 이름을 바꿔 정치 루프 안에 넣는다.
